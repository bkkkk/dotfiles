# Plan: Sync Claude Configs via Dotfiles

## Goal

Track Claude Code configuration files in this repo and symlink them into `~/.claude/` on install — consistent with how zsh, mise, and git configs are already managed here.

## What to Sync

| File / Dir | Location in dotfiles | Notes |
|---|---|---|
| `CLAUDE.md` | `setups/claude/CLAUDE.md` | Global Claude instructions |
| `settings.json` | `setups/claude/settings.json` | Model, plugins, marketplace config |
| `commands/` | `setups/claude/commands/` | Custom slash command definitions |
| `agents/` | `setups/claude/agents/` | Custom sub-agent definitions |

### What NOT to sync

| File / Dir | Why |
|---|---|
| `skills/` | Marketplace-managed. `xclaude-plugin-marketplace` (configured in `settings.json`) auto-downloads and updates these. Committing them would conflict with auto-updates and bloat the repo. Syncing `settings.json` is sufficient — skills reinstall automatically on a new machine. |
| `projects/` | Per-project memory, conversation history, and task state. Private and machine-specific. |
| `history.jsonl`, `daemon.*`, `cache/` | Runtime files generated by Claude Code. Never tracked. |

## Directory Structure (after implementation)

```
setups/
  claude/
    claude.sh           # install script (creates symlinks)
    CLAUDE.md           # global Claude instructions
    settings.json       # model + plugin preferences
    commands/           # custom slash commands
      build.md
      create-view.md
      ...
    agents/             # custom agent definitions
      ios-architect.md
      ios-researcher.md
      ...
```

## Symlink Mapping

```
~/.claude/CLAUDE.md     → ~/dotfiles/setups/claude/CLAUDE.md
~/.claude/settings.json → ~/dotfiles/setups/claude/settings.json
~/.claude/commands/     → ~/dotfiles/setups/claude/commands/
~/.claude/agents/       → ~/dotfiles/setups/claude/agents/
```

Directories are symlinked as a whole so any new command or agent added to the directory is automatically tracked in git.

## Install Script (`setups/claude/claude.sh`)

The script should:

1. Create `~/.claude/`
1. Detect if `~/.claude/<target>` already exists as a non-symlink
2. Back it up to `~/.claude/<target>.bak` before replacing
3. Create the symlink with `ln -sf`

## Wiring into `install.sh`

Add a call to `setups/claude/claude.sh` in `install.sh` alongside the existing brew/zsh/mise steps.

## Open Questions / Decisions Still Needed

1. **New commands and agents added post-install:** Since directories are symlinked, any file Claude Code writes into `~/.claude/commands/` or `~/.claude/agents/` lands directly in the dotfiles repo. Is that the desired behavior, or should individual files be symlinked instead of the whole directory?

Yes, that's the point.

2. **`settings.json` has auto-update behavior:** The `xclaude-plugin-marketplace` entry has `"autoUpdate": true`. Claude Code may rewrite `settings.json` when it adds or updates plugins. Since `settings.json` will be a symlink, those writes go directly into the dotfiles file — which is fine, but means you may see unexpected `git diff` noise after running Claude. Acceptable?

That's actually a feature. We want to sync changes into git.

3. **Custom skills (non-marketplace):** If you ever write a local skill that lives in `~/.claude/skills/` but is NOT from the marketplace, it won't be tracked. We could add a `setups/claude/skills/` directory and symlink it, but only if you have or plan to have hand-written skills there.

No need for now.

4. **Should `claude.sh` be called automatically from `install.sh`**, or do you want it as an opt-in separate step (like `macos.sh`)?

Actually `macos.sh` and `claude.sh` should both be in `install.sh`. `macos.sh` being left out is a bug. Address it as well.
